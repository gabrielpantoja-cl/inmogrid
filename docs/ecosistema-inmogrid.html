Frontend — lo que se ve

Vercel — La vitrina del negocio

Aquí vive el código de la página web (Next.js). Vercel es como un dueño de tienda que tiene su local en la nube: cada vez que alguien entra a inmogrid.cl, Vercel le entrega las páginas al instante desde servidores distribuidos por todo el mundo.

Cuando Gabriel sube cambios al código en GitHub, Vercel los detecta y actualiza el sitio automáticamente en segundos, sin apagarlo.

Next.js 15 React 19 TypeScript Tailwind CSS Auto-deploy desde GitHub
🗄️
Base de datos + Autenticación

Supabase — La bodega y el portero

Hace dos cosas fundamentales:

1. Guarda todo — perfiles, publicaciones, conexiones entre usuarios, mensajes. Es la bodega donde vive la información.

2. Controla quién entra — cuando haces "Iniciar sesión con Google", Supabase es el portero que verifica tu identidad y te entrega una llave de acceso.

Detalle importante: esta misma bodega la comparte con pantojapropiedades.cl durante la transición.

PostgreSQL Google OAuth Prisma ORM Tiempo real
⚙️
Automatizaciones — trabajo entre bambalinas

VPS Oracle — El asistente nocturno

Un servidor propio (siempre encendido) que corre tareas automáticas que no necesitan que el usuario las active. Por ejemplo: enviar correos, procesar datos, conectar servicios externos. Vive en api.inmogrid.cl y usa N8N como motor de automatizaciones (tipo Zapier pero privado).

N8N Workflows Docker api.inmogrid.cl

Proveedor de identidad

Google — El documento de identidad

inmogrid.cl no guarda contraseñas. En cambio, cuando inicias sesión, le pregunta a Google "¿es realmente esta persona?". Google confirma, y Supabase te deja entrar. Es como mostrar tu carnet en la puerta.

Vercel = La tienda

El local que el cliente visita. Siempre abierto, rápido, y se renueva solo cuando hay cambios.
